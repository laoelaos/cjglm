- **位置**：§4 T17「建议测试数据」段落
- **问题描述**：产出建议 Mat4x4→Mat4x3 行收缩测试以 `Mat4x4(Float32(1..16))` 为源数据，预期结果为 `Vec3(1,2,3)、Vec3(4,5,6)、Vec3(7,8,9)、Vec3(10,11,12)`。但 Mat4x4 采用列主序布局（`type_mat4x4.cj:13-18`）：列 0=Vec4(1,2,3,4)、列 1=Vec4(5,6,7,8)、列 2=Vec4(9,10,11,12)、列 3=Vec4(13,14,15,16)。行收缩（R=4→3）应截断每列末行：`Vec3(1,2,3)`、`Vec3(5,6,7)`、`Vec3(9,10,11)`、`Vec3(13,14,15)`。产出建议的 `Vec3(4,5,6)` 等值暗示了行主序理解，执行者依据此建议编写的测试将产生错误断言。
