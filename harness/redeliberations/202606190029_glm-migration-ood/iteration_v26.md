## 需要解决的问题（RETRY）

- **问题描述**：`fromBoolVec` 定义为扩展成员函数导致调用方需持有无关 dummy 实例作为 receiver，与本设计其他工厂函数风格不一致
- **所在位置**：§4.8 `fromBoolVec` 签名清单
- **严重程度**：一般
- **改进建议**：改为包级独立函数或 VecN 静态函数，消除 dummy instance 依赖

- **问题描述**：Vec4 多重重载构造函数在重载解析中存在歧义风险，影响范围约 30 个构造函数签名，重构成本高；§10 未新增重载解析验证项，未定义回退方案
- **所在位置**：§4.1 "跨 Q 构造函数重载解析风险与约束验证"段落末
- **严重程度**：一般
- **改进建议**：在 §10 新增重载解析验证项；定义具体回退方案（移除歧义分支并转为具名工厂函数）；前置到 §2.1 编码前验证清单

- **问题描述**：`equalEpsilon` 定义于 `extend` 块中（非 const），在 D29 回退路径描述段落中未显式标注 const 限制，编码实施者可能遗漏
- **所在位置**：§4.5 D29 回退描述段落
- **严重程度**：一般（中等）
- **改进建议**：在 D29 回退描述段落添加显式 const 限制警示；在 §12.2 验收标准中新增 D29 回退下 const 上下文容差比较的备选方案验证项
