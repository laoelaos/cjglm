## 问题清单

### 问题 1（严重）：§4.3 Vec2 算术运算符模板中 `%` 运算符标量版本存在代码实现错误

**问题描述**：§4.3 "`Vec2<T,Q>` 算术运算符完整语法模板"中标量右操作数 `%` 运算符的代码实现存在两处事实错误：
- 参数 `rhs` 类型为标量 `T`（`operator func %(rhs: T)`），但模板中第二个分量使用了 `this.y / rhs`（用 `/` 替代了 `%`），应为 `this.y % rhs`

错误代码（约第 1040 行）：
```
Vec2(this.x % rhs, this.y / rhs)
```
应为：
```
Vec2(this.x % rhs, this.y % rhs)
```

**所在位置**：§4.3 Vec2 算术运算符模板（约第 1039-1041 行）

**严重程度**：严重

**改进建议**：将 `this.y / rhs` 修正为 `this.y % rhs`。同步检查 Vec1/Vec3/Vec4 的 `%` 运算符模板中是否存在同样的复制遗漏。

**落地影响**：此模板被标注为"可作为其他 VecN 运算符实现的编码参考模板"，实施者若直接复制将产生逻辑错误（`/` 替代 `%`）。Vec1/Vec3/Vec4 实施者若以 Vec2 为蓝本扩展分量数，会进一步传播此错误。

---

### 问题 2（一般）：const 上下文 API 可用性速查表中 `equalExact` 的 const 替代方案指引不准确

**问题描述**：§3.2 const 上下文 API 可用性速查表（约第 449 行）中 `v.equalExact(other)` 行的 const 替代方案包含"改用包级独立函数"，但当前设计文档中不存在包级独立的 `equalExact` 函数。§4.5 明确 `equalExact` 的 const 化方案为"将定义移至 struct 体内声明为 `const` 实例成员函数（D33 备选方案）"，不存在"包级独立函数"替代路径。

**所在位置**：§3.2 const 上下文 API 可用性速查表最末行（约第 449-455 行）

**严重程度**：一般

**改进建议**：移除"包级独立函数"字样，统一为与 §4.5 一致的描述："在 struct 体内自行定义 const 版本的 `equalExact`（D33 备选方案）或手动编写精确内联比较表达式"。

### 问题 3（轻微）：§1 版本谱系说明与修订说明存在断档

**问题描述**：§1 版本谱系说明（约第 8 行）声明"版本追溯路径：v4 → v5 → v6 → v7 → v8 → v9 → v10 → v11 → v12 → v13 → v14"，跳过了 v1-v3。但文件末尾的修订说明包含 v2、v3、v4 的完整修订记录，且 §1 历史问题追踪表（P5）引用"v2 已解决"版本标识问题。这使追溯路径与修订记录之间存在认知断档——实施者需要自行推断 v1-v3 是早期版本且已被 v4 重构覆盖。

**所在位置**：§1 文件头部版本谱系说明 + 文件末尾修订说明章节

**严重程度**：轻微

**改进建议**：在版本谱系说明中补充说明，如"追溯路径从 v4 起始（v1-v3 已被 v4 重构合并，不再追溯）"，以消除修订记录与谱系说明的不一致。

### 问题 4（一般）：§4.3 Vec2 算术运算符模板中 `%` 参数命名与类型存在易混淆问题

**问题描述**：§4.3 Vec2 算术运算符模板中，标量版本 `%` 的签名定义为 `operator func %(rhs: T)`，而 Vec-Vec 版本定义为 `operator func %(rhs: Vec2<T, Q>)`。模板内部使用 `this.x % rhs` 对整数 `T` 正确。但模板注释声称"可作为其他 VecN 运算符实现的编码参考模板"，而 Vec1/Vec3/Vec4 的 `%` 在整数 `T` 上可编译通过，浮点 `T` 上在实例化处产生编译错误——此行为已有设计覆盖，但模板本身未标注此约束，实施者若直接复制到 Vec1/Vec3/Vec4 扩展时可能误以为模板无条件正确。

**所在位置**：§4.3 Vec2 算术运算符模板注释（约第 999 行）声称"可作为其他 VecN 运算符实现的编码参考模板"，但未标注 `%` 的整数约束

**严重程度**：轻微

**改进建议**：在模板注释中补充标注，如"注：`%` 仅对整数 `T` 可用，浮点 `T` 在实例化时产生编译错误（D5 延迟检查语义）"

### 异常场景与边界条件评估

§9 异常场景和边界条件补充涵盖了以下维度：

- 除零行为（§9.2）：整数/浮点区分明确
