   - 改进建议：新增"编码实现效率建议"提示框，推荐多光标编辑、片段模板或脚本生成；在 §8.3 迁移顺序中为每个子步骤增加预期代码量标注。

4. **[一般]** P37 的整体风险评级与后续轮次依赖关系存在隐性矛盾："无高等级风险文件"的评估因未经逐 API 验证而本质上是推测性的；第 2 轮编码阶段补充 API 映射时，可能发现遗漏需回溯修改首轮代码。
   - 改进建议：修正措辞消除"结论已确认"暗示；在 §8.3 阶段 1 备注列补充 shim 文件 API 覆盖评估为依赖项级别而非逐 API 级别。

5. **[轻微]** §1 确定性声明的"核心假设"清单与 §2.1 验证项的高影响项不完全一致：Z 类别阈值包含②（中影响）和⑮（低影响）却排除了④（高影响），缺乏自洽的分类依据。
   - 改进建议：将 Z 类别阈值修正为与 §2.1 影响范围分类一致，或明确说明包含低影响项 ⑮ 的理由。
