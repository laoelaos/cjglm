### 1. §3.2 const 上下文 API 可用性速查表的事实错误未被修复（中等）
`v.increment()` 行的 const 替代方案仍标注为"改用 `v = v + T(1)` 或 `add(T(1), v)` 等展开模式"。其中 `v = v + T(1)` 包含赋值操作 `=`——赋值在 const 函数体内不可用（const 函数不可修改实例成员），因此该替代方案本身不是 const 兼容的。正确的 const 替代方案应为 `add(T(1), v)`（包级独立函数，声明为 const）或直接构造新向量。此问题在上一轮审查（b_v11_diag_v2.md §2.3）中已被指出，但 v11 修订未修复。

**改进建议**：将 `v.increment()` 行的 const 替代方案中的 `v = v + T(1)` 移除，仅保留 `add(T(1), v)`（包级独立函数，声明为 const）作为正确的 const 兼容替代方案。同步检查该速查表中其他行的替代方案是否也存在类似问题（如 `v.decrement()` 行）。
