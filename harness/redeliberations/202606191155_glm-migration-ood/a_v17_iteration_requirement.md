### 问题 2（一般）：§3.2 const 上下文 API 可用性速查表中 `decrement()` 的 const 替代方案存在语义错误
`v.decrement()` 行的 const 替代方案标注为"改用 `sub(T(1), v)`"，但 `sub(s: T, v: VecN<T,Q>)` 的语义定义为标量减向量（`s - v`），计算的是 `1 - v_i`，而非 decrement 所需的 `v_i - 1`。
- 所在位置：§3.2 const 上下文 API 可用性速查表（第 477 行）
- 改进建议：修正为"改用 `v - T(1)`（若二元运算符 `-` 可声明为 const）"或"改用 `v + (-T(1))` 通过加法实现递减效果"。

### 问题 3（一般）：`@OverflowWrapping` 标注对浮点运算符代码生成的潜在影响缺乏验证覆盖
§4.6（第 1229 行）明确承认 `@OverflowWrapping` 在浮点类型上可能"抑制编译器对 NaN/Infinity 检测的优化路径"，但 §2.1 编码前验证清单的 25 项验证中无任何一项针对此风险设计验证方法。若编码阶段才暴露，将导致 §4.3 全部 20 个浮点算术运算符的标注策略需要从"统一标注"改为"条件性标注"。
- 所在位置：§4.6（第 1229 行）浮点 `@OverflowWrapping` 说明段、§2.1 编码前验证清单
- 改进建议：在 §2.1 编码前验证清单中新增验证项，编写原型测试对比标注与不标注 `@OverflowWrapping` 时含 NaN/Infinity 分量的浮点运算结果差异；若发现语义差异，制定条件性标注策略（仅整数 T 标注）。

### 问题 4（轻微）：§7 D6 `GLM_CONFIG_CLIP_CONTROL` 常量的 "已验证结论" 措辞与 §8.4 验证项②的依赖关系未显式关联

- 所在位置：§7 D6（第 1417 行）默认值推导结论段落、§8.4（第 1894 行）前置验证项①
- 改进建议：在 §7 D6 "已验证正确" 措辞后添加指向 §8.4 的交叉引用脚注。

### 问题 5（轻微）：§8.3 阶段 0 "非 Vec 文件 API 快速扫描" 与 P37 的延迟对照存在执行时序矛盾
