## 需要解决的问题（仅 RETRY 时存在）

- **问题描述**：行向量×矩阵乘法签名错误。设计方案签名 `Vec{C}<T,Q> * Mat{C}x{R}<T,Q> → Vec{R}<T,Q>` 与 GLM 参考实现不符，正确签名应为 `Vec{R}<T,Q> * Mat{C}x{R}<T,Q> → Vec{C}<T,Q>`。
- **所在位置**：§3.5「二元运算符（矩阵-向量）」
- **严重程度**：严重
- **改进建议**：修正为 `Vec{R}<T,Q> * Mat{C}x{R}<T,Q> → Vec{C}<T,Q>`，并在 §2 矩阵类型概览表中补充示例说明。

- **问题描述**：跨尺寸矩阵乘法运算符未完整定义，设计方案仅给出一个跨类型示例，缺少完整的组合清单或系统性匹配规则。
- **所在位置**：§3.5「二元运算符（矩阵-矩阵，同尺寸）」— 矩阵乘法子节
- **严重程度**：一般
- **改进建议**：补充跨尺寸矩阵乘法完整的组合规则说明，按"左矩阵列数 = 右矩阵行数则形态兼容，结果尺寸为 (左矩阵行数 × 右矩阵列数)"的规则对所有 9 个矩阵类型列出其乘法伙伴集。

- **问题描述**：跨矩阵类型转换构造的填充规则描述过于模糊（"额外值填默认值（如 0 或 1）"），未给出单位矩阵扩展的精确语义。
- **所在位置**：§3.3「构造函数体系」第 8 条（跨矩阵类型转换构造）
- **严重程度**：一般
- **改进建议**：补充明确填充规则：缺失的列/行补齐为单位矩阵的相应列/行（即缺失元素中 `(i,i)` 填 `T(1)`，其余填 `T(0)`）。

- **问题描述**：ext/ 别名文件数量描述前后矛盾，§2 模块清单列 18 个文件，§3.8 称"16 个矩阵别名文件"。
- **所在位置**：§2 模块清单（第 64-73 行）vs §3.8（第 287-289 行）
- **严重程度**：一般
- **改进建议**：统一为 18 个文件（9 尺寸 × 2 精度），删除"精度不重复的若干组合"的模糊描述。

- **问题描述**：矩阵除法 `/` 的 stub 依赖表述不准确，未区分 `mat / mat`（依赖 inverse）与 `mat / scalar`（不依赖 inverse），且 `this` 的成员函数风格在仓颉 `extend` 块中不准确。
- **所在位置**：§3.5「二元运算符（矩阵-矩阵，同尺寸）」— 除法子节
- **严重程度**：一般
- **改进建议**：明确区分 `mat / scalar`（本阶段可实现，逐元素操作）和 `mat / mat`（依赖 inverse，推迟至阶段四），将 `this * inverse(rhs)` 改为 `mat *= inverse(rhs)`。

- **问题描述**：标量-矩阵除法的 stub 策略未在 §5 中说明，仅提及矩阵除法依赖 inverse 为 stub，可能误导开发者将所有除法形式标记为 stub。
- **所在位置**：§5「错误处理策略」
- **严重程度**：轻微
- **改进建议**：在 §5 中明确注明仅 `mat / mat` 受 stub 影响，`mat / scalar`、`scalar / mat` 可本阶段完整实现。
