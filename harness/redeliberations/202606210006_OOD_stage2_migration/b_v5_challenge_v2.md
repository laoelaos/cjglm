## 逐维度审查

### 1. 证据充分性

**[问题-严重]** P4 的核心反例所引用的 GLM 源码存在严重事实错误。报告 P4 声称 GLM 的 Mat4x4 ← Mat4x2 转换构造中第 3 列为 `col_type(m[3], 0)`——意即 GLM 从源矩阵 Mat4x2 的第 3 列构造 Vec4 并在末尾补 0，从而使 `(3,3)` 位置填 `T(0)` 而非设计规则预测的 `T(1)`。但经直接核对 GLM 源码 `type_mat4x4.inl:246-257`，Mat4x4 ← Mat4x2 的实际实现为：

```
value{col_type(m[0], 0, 0), col_type(m[1], 0, 0), col_type(0, 0, 1, 0), col_type(0, 0, 0, 1)}
```

第 3 列是 `col_type(0, 0, 0, 1)`，**而非** `col_type(m[3], 0)`。`(3,3)` 位置的值是 `1`（单位矩阵对角线），与设计规则预测的 `T(1)` **完全一致**。报告中 P4 的反例验证表声称 `Mat4x4 ← Mat4x2` 在 `(3,3)` 处 `GLM 实际 = T(0)` 且标记 `✗`，这是基于虚构证据得出的虚假反例。

与此同时，P4 报告将 Mat4x4 ← Mat4x3 的实现（`col_type(m[3], 1)`）与 Mat4x4 ← Mat4x2 的实现混淆，声称后者也有 `col_type(m[3], ...)` 形式。这是两个完全不同的转换构造函数，不能混为一谈。

此事实错误直接动摇 P4 的核心结论。P4 声称的"填充规则与 GLM 不一致"在 Mat4x4 ← Mat4x2 这一关键反例上不成立。

**[问题-严重]** 承上，P4 虽然核心反例有误，但存在一个未被报告准确识别的真实差异：设计中 §3.3 第 8 条的填充规则声明"源矩阵 M_src[j][i] 复制到目标矩阵的相同坐标位置，范围 0 ≤ j < min(C_src, C_dst)、0 ≤ i < min(R_src, R_dst)"。对于 Mat4x4 ← Mat4x2，min(C_src, C_dst) = 4, min(R_src, R_dst) = 2，因此设计规则预测第 2、3 列的前 2 个行元素（src[2][0], src[2][1], src[3][0], src[3][1]）应被复制到目标矩阵。但 GLM 实际行为是第 2、3 列**完全未从源矩阵取值**——而是直接从单位矩阵构造（`col_type(0, 0, 1, 0)`、`col_type(0, 0, 0, 1)`）。这是一个真实的设计偏差，但 P4 报告未能将其正确识别和定位，反而给出了错误的反例和错误的分析表格。P4 的正确问题在于设计规则对"复制范围"的定义——当 C_src ≥ C_dst 时（即目标矩阵没有额外列），设计规则的 `(j,i)` 坐标复制是否会不恰当地保留源矩阵中行维度不足的列的部分数据，这是一个需要精确界定的语义问题，而非 P4 所声称的"单位矩阵扩展填充不一致"。

**[通过]** P1 对上一轮审查 P1（填充规则与 GLM 不一致）的纠正是有效的。经核对 `type_mat4x4.inl` 中 8 种转换构造函数，GLM 在所有情况下确实采用单位矩阵扩展填充（对角线位置填 1、其余填 0），而非全零填充。P1 的源码引用和逐例验证具有充分证据。

**[通过]** P2 对 `init()` 默认单位矩阵构造函数在无约束泛型 T 下不可行的分析逻辑自洽、证据充分。P2 准确识别了 §3.3 第 1 条的核心问题——`T(0)/T(1)` 在无约束泛型 T 上不可构造——并正确合并了上一轮 P4/P9 的矛盾。

**[通过]** P7 对需求响应覆盖度的补充分析有效，识别了 §3.3 第 1 条未显式关联 DV-01 约束的遗漏。

### 2. 逻辑完整性

**[问题-严重]** P1 与 P4 之间存在逻辑矛盾。P1 花费大量篇幅论证"当前设计的填充规则与 GLM 参考实现一致"并据此撤回上一轮 P1，而 P4 又声称"设计的通用填充规则描述与 GLM 的实际逐列构造行为不完全一致"。两个问题对同一填充规则给出了相互否定的结论。产出作者无法同时采纳 P1（确认填充规则一致）和 P4（声称填充规则不一致）。

进一步分析：P1 论证的"一致性"实际上是正确的——即在"额外行列从单位矩阵取值"的语义层面，设计与 GLM 一致。P4 所发现的差异并非填充值本身的问题，而是设计规则中"源矩阵元素复制范围"的定义问题（当源矩阵列数 ≥ 目标矩阵列数但行数 < 目标矩阵行数时，如何处理源矩阵中行维度不足的列）。P4 混淆了两个不同的问题，导致其结论与 P1 矛盾。

**[通过]** P2、P3、P5~P8 各问题之间无逻辑矛盾，改进建议与问题一致。P2 合并了上一轮 P4/P9 的矛盾后逻辑自洽。

**[通过]** 修订说明（v2）中对上一轮质询意见的逐条回应逻辑清晰、依据充足。

### 3. 覆盖完备性

**[问题-一般]** P4 虽然证据有误，但确实触及了一个设计规则的真实歧义点——设计 §3.3 第 8 条的坐标复制范围定义 `0 ≤ j < min(C_src, C_dst)、0 ≤ i < min(R_src, R_dst)` 在某些转换场景下的语义是否与 GLM 逐列构造行为一致。报告应当正确识别并精确描述此差异（源矩阵列数充足但行数不足时，GLM 的逐列构造方式可能导致部分列完全从单位矩阵取值而非按坐标复制），而非基于错误的源码引用构建反例。当前 P4 的覆盖是有的，但精度严重不足。

**[通过]** 报告覆盖了需求响应充分度（P7）、事实正确性（P1、P4、P6）、逻辑一致性（P3）、完整性（P2、P5、P8）等维度。历史迭代反馈的追踪覆盖情况在修订说明中已充分说明。

**[通过]** P5 补充了上一轮 P11（ext/ 向量别名与 fwd.cj 别名定位差异）未获回应的问题。

### 4. 报告必要性

**[问题-一般]** P6（`type_mat4x3.cj ★ — Mat4x4<T,Q>` 笔误）属文档校对类细节，从编码落地角度影响极低（编码者不会仅凭概览表中类型名写代码），且审查任务侧重"需求响应充分度、整体深度和完整性"。上一轮质询已指出可降级，报告仍保留并提出"可能导致编码者误命名"的过度推断，必要性不足。建议删除或降级为附注。

**[通过]** P8（`[]` 和 `col()` 在 const 上下文中的可行性前瞻）属于对设计完整性的合理补充，虽不阻塞当前阶段但涉及需求响应完整性，必要性可接受。

## 质询要点

- **问题1**：P4 的核心反例——Mat4x4 ← Mat4x2 的 GLM 源码引用 `col_type(m[3], 0)` 为虚构，实际应为 `col_type(0, 0, 0, 1)`。基于此虚构证据得出的反例验证表（声称 `(3,3)` 处 GLM 填 `T(0)`）完全错误。P4 的核心结论"设计填充规则与 GLM 不一致"在 Mat4x4 ← Mat4x2 这一关键反例上不成立
  - **原因**：错误的事实基础导致错误的问题定位。产出作者若据 P4 修改填充规则，会引入不必要的偏离
  - **建议方向**：重新核对 GLM 所有 8 种 Mat4x4 转换构造函数的源码，正确识别设计规则与 GLM 行为的真实差异点——即坐标复制范围的语义定义（源矩阵列数 ≥ 目标列数但行数不足时，部分列的数据是否保留）。注意 Mat4x4 ← Mat2x4（源矩阵列数不足，额外列完全从单位矩阵取值）与 Mat4x4 ← Mat4x2（源矩阵列数充足但行数不足，GLM 的部分列也完全从单位矩阵取值）是两种不同的扩展场景，需要分别验证

- **问题2**：P1 与 P4 对同一填充规则给出互相否定的结论——P1 确认"与 GLM 一致"，P4 声称"与 GLM 不完全一致"
  - **原因**：产出作者无法同时采纳两个互相矛盾的审查意见，严重影响审查报告的可操作性
  - **建议方向**：统一 P1 和 P4 的结论。真实情况是：填充值策略（对角线填 1、其余填 0）在大多数情况下与 GLM 一致（P1 正确），但设计规则中坐标复制范围的定义与 GLM 逐列构造行为在特定场景（如 C_src ≥ C_dst 但 R_src < R_dst）下存在差异（P4 的方向正确但证据错误）。应将 P4 重写为精确描述此差异，而非推翻 P1

- **问题3**：P6 笔误问题必要性不足
  - **原因**：文档校对类细节与任务审查视角的关联度极低
  - **建议方向**：删除 P6 或降级为附注
