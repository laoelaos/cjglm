## 需要解决的问题

- **问题描述**：跨尺寸乘法兼容性表遗漏 Mat3x3*Mat2x3 和 Mat4x3*Mat2x4 两个合法乘法组合，跨尺寸乘法总数应为26而非24，总有效重载应为29而非27
- **所在位置**：§3.5 跨尺寸乘法兼容性表；D04；§8 产出物清单中相关数量描述
- **严重程度**：严重
- **改进建议**：补充遗漏的2个乘法组合，同步更新跨尺寸乘法总数和总有效重载数量

- **问题描述**：非方阵 operator/=(scalar) 复合赋值运算符的存在性在§3.5表格中未明确，与方阵 /=(mat/mat) stub状态区分模糊
- **所在位置**：§3.5 复合赋值运算符表格第②类和第④类
- **严重程度**：一般
- **改进建议**：在第②类中明确 /=(scalar) 对所有9种矩阵类型均可用，补充说明方阵同时存在两版 /= 重载按右操作数类型选择

- **问题描述**：/=(scalar) 与 /=(mat/mat) 的 stub 差异在表格中未显式标注，可能导致编码者误将 /=(scalar) 也标记为 stub
- **所在位置**：§3.5 复合赋值运算符表格；§5 错误处理策略
- **严重程度**：轻微
- **改进建议**：在第②类中注明 /=(scalar) 逐元素操作不依赖 inverse 为完整实现（非stub），与第④类方阵 /=(mat/mat) stub 形成显式区分

- **问题描述**：fromMat 6a/6b 中 one 参数在目标尺寸小于源尺寸时冗余但不可省略，未分析此场景
- **所在位置**：§3.3 第 6a/6b 条
- **严重程度**：一般
- **改进建议**：明确说明 one 参数在尺寸缩减方向上的冗余性，评估是否可额外提供无 one 参数的缩减方向工厂函数重载

- **问题描述**：第7条 fromMat<U,P> 与第6a条在源矩阵与目标矩阵元素类型相同且尺寸相同时功能完全重叠，未说明调用优先级
- **所在位置**：§3.3 第7条 vs 第6a条
- **严重程度**：一般
- **改进建议**：明确说明第7条在同类型同尺寸场景下冗余，推荐调用方优先使用第6a条

- **问题描述**：outerProduct 参数 c 被命名为"列向量"可能误导编码者认为 c 分量数=列数，实际 c 分量数=行数
- **所在位置**：§3.7 matrix.cj — outerProduct 签名列表
- **严重程度**：一般
- **改进建议**：明确 outerProduct 参数语义：第一个参数 c 分量数=结果矩阵行数R，第二个参数 r 分量数=结果矩阵列数C；补充说明 GLM 中 c 指"用于构建列"的向量
