## 需要解决的问题

- **问题描述**：fwd.cj 缺少 Float32/Float64 以外精度族的矩阵别名（fmat/f32mat/f64mat 系列及 dmat 非方阵全尺寸系列），且非方阵默认精度别名（mat2x3~mat4x3 等）未显式列出，纳排决策缺失
- **所在位置**：§8 修改文件 — fwd.cj 矩阵别名命名规则；§3.8 ext/ 别名文件
- **严重程度**：一般
- **改进建议**：逐一对照 C++ GLM fwd.hpp 列出全部矩阵别名族，明确纳入与排除范围及理由；补充 dmat 非方阵别名的纳入决策

- **问题描述**：非方阵复合赋值运算符 `*=(mat)` 适用范围不清晰，"由编译器自动生成"措辞可能误导编码者认为所有矩阵类型均获得矩阵乘复合赋值
- **所在位置**：§3.5 复合赋值运算符段落
- **严重程度**：一般
- **改进建议**：明确仅方阵类型（Mat2x2/Mat3x3/Mat4x4）自动获得 `*=(same_mat)`；注明非方阵因返回类型不同无法自动生成；注明方阵 `*=(mat)` 自动生成语义与 C++ GLM 显式定义一致

- **问题描述**：跨矩阵类型转换构造中"对应位置"语义描述不精确，未给出 (row, col) 坐标对应的精确定义
- **所在位置**：§3.3 构造函数体系第 8 条
- **严重程度**：一般
- **改进建议**：补充坐标对应精确定义：源矩阵 M[j][i] 复制到目标矩阵相同坐标，范围 0 ≤ j < min(C_src, C_dst)、0 ≤ i < min(R_src, R_dst)；额外行列从单位矩阵取值

- **问题描述**：`scalar / mat` 逐元素除法不依赖 inverse、可本阶段实现，但 §5 错误处理策略中遗漏
- **所在位置**：§3.5 标量-矩阵运算；§5 错误处理策略
- **严重程度**：一般
- **改进建议**：在 §3.5 明确 div(s, m) 语义为逐元素 s / each_element；在 §5 补充 scalar / mat 可本阶段完整实现

- **问题描述**：`mat / mat` 和 `mat / vec` 除法运算符仅方阵有定义，设计文档"mat / mat"措辞可能误导编码者为所有9种矩阵类型实现该运算符
- **所在位置**：§3.5 二元运算符矩阵-矩阵除法子节
- **严重程度**：一般
- **改进建议**：明确指出 mat/mat 和 mat/vec 仅在方阵类型上定义，非方阵不提供

- **问题描述**：lib.cj 对 scalar_mat_ops.cj 全局函数的导出使用"建议"措辞，编码者无法确定是否应导出
- **所在位置**：§8 修改文件 — lib.cj 条目
- **严重程度**：一般
- **改进建议**：将"建议同步导出"改为明确决策；确认 lib.cj 现有 public import 已自动覆盖同包同名函数

- **问题描述**：27个跨尺寸矩阵乘法重载的文件归属策略未定义，直接影响编码落地
- **所在位置**：§3.5 跨尺寸乘法兼容性表；§7 D04
- **严重程度**：一般
- **改进建议**：明确跨尺寸乘法运算符定义在左操作数矩阵类型的 extend 块中，在对应 type_mat{C}x{R}.cj 文件顶部 import 所需右操作数类型

- **问题描述**：`[]` 运算符取值版本返回列向量副本，不支持分量级链式修改，但 §9 差异声明中未记录此用户可见差异
- **所在位置**：§3.4 行列访问；§9 差异声明
- **严重程度**：一般
- **改进建议**：在 §9 差异声明新增条目：`[]` 取值版本返回副本，仅可整列赋值 `m[0] = new_col`，不能直接修改分量 `m[0].x = value`
