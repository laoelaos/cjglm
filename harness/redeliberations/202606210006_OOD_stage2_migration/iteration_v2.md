## 需要解决的问题（仅 RETRY 时存在）

- **问题描述**：ext/ 向量别名文件未纳入设计范围，路线图明确将其列为阶段二范围
- **所在位置**：§2「包组织」— package glm.ext 文件清单
- **严重程度**：严重
- **改进建议**：明确阶段二是否包含 ext/ 向量别名文件的创建；若包含补充对应向量别名文件，若不包含说明理由并在 §9 偏差声明中记录

- **问题描述**：D08 等效替代建议数学错误——`+= identity` ≠ `++m`，两者数学语义完全不同
- **所在位置**：§7 D08（决策表第 8 行）
- **严重程度**：严重
- **改进建议**：修正为"可通过 `+= T(1)`（逐元素加 1）等价替代"或标注行为差异

- **问题描述**：跨矩阵类型转换构造实际需 72 个独立构造函数重载，设计未评估实现量级可行性和缩减方案
- **所在位置**：§3.3「构造函数体系」第 8 条
- **严重程度**：严重
- **改进建议**：明确 72 个重载是否可接受，若不能提出缩减方案；为每个转换构造函数明确填充规则边界情况；评估仓颉重载解析歧义问题

- **问题描述**：`componentAt` 返回列向量，与 Vec 版本返回标量分量的语义矛盾
- **所在位置**：§3.4「行列访问」
- **严重程度**：一般
- **改进建议**：改名如 `col(i: Int64)` 避免歧义，或改为返回标量元素 `componentAt(col, row)` 与 Vec 语义对齐

- **问题描述**：`row()` 成员函数在本阶段无实际作用，且后续替换为自由函数时产生 API 不兼容
- **所在位置**：§3.4「行访问函数」
- **严重程度**：一般
- **改进建议**：明确决策——要么作为阶段二成员函数实现，要么标注为延期至阶段四随 matrix.cj 提供自由函数版本；更新 §7 决策表

- **问题描述**：跨类型构造函数（不同 T）未明确覆盖跨类型同尺寸矩阵转换
- **所在位置**：§3.3「构造函数体系」
- **严重程度**：一般
- **改进建议**：补充跨类型同尺寸转换构造函数签名，或说明已包含在其他条款的泛化描述中并明确签名

- **问题描述**：`scalar_mat_ops.cj` 中 36 个全局函数的溢出策略未说明
- **所在位置**：§5「错误处理策略」
- **严重程度**：一般
- **改进建议**：明确 `scalar_mat_ops.cj` 各函数的 `@OverflowWrapping` 标注策略，与阶段一 `scalar_vec_ops.cj` 保持一致

- **问题描述**：`mat / vec` 和 `vec / mat` 除法未讨论，它们依赖 inverse 实现
- **所在位置**：§3.5「二元运算符（矩阵-矩阵）」除法子节
- **严重程度**：一般
- **改进建议**：明确 `mat / col_vec` 和 `row_vec / mat` 是否在本阶段支持；若依赖 inverse 也推迟并注明
