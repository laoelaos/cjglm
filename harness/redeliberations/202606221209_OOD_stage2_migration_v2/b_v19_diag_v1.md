## 发现的待解决问题

### 问题 1：B 类方向计数错误（§3.3 行扩展规则修正说明）

| 项目 | 内容 |
|------|------|
| **问题描述** | 文档在"行扩展规则修正说明"段落中称"**8 个 B 类方向**中除 Mat4x4←Mat4x2 外，**其余 7 个**均遵循"。但根据 §3.3 9×9 转换矩阵表统计，B 类方向（C 不变、仅 R 扩展）共 **9 个**：Mat2x2→Mat2x3、Mat2x2→Mat2x4、Mat2x3→Mat2x4、Mat3x2→Mat3x3、Mat3x2→Mat3x4、Mat3x3→Mat3x4、Mat4x2→Mat4x3、Mat4x2→Mat4x4（DEVIATION）、Mat4x3→Mat4x4。其中 8 个遵循主规则，1 个偏差（Mat4x4←Mat4x2）。应修正为"9 个 B 类方向中除 Mat4x4←Mat4x2 外，其余 8 个均遵循"。 |
| **所在位置** | §3.3，"行扩展规则修正说明"段落（L333） |
| **严重程度** | 中等 — 算术错误虽不影响功能理解，但可能在后续编码验证计划的 B 类方向统计中造成混淆 |
| **改进建议** | 将"8 个 B 类方向"修正为"9 个 B 类方向"，将"其余 7 个"修正为"其余 8 个"。同时检查全文其他提及 B 类方向数量的地方（如"B 类方向与 GLM .inl 逐项对照分析"表头、B 类方向单元测试验证要求），确认已正确计数或修正为一致表述。 |

### 问题 2：转换矩阵表中 Mat4x2→Mat4x3 标签缺少 "B:" 前缀

| 项目 | 内容 |
|------|------|
| **问题描述** | §3.3 fromMat 6a 9×9 转换矩阵表中，Mat4x2 行→Mat4x3 列的标签为 `rowExt(3)`，缺少 `B:` 前缀。该方向属于 B 类（C=4→4 不变、R=2→3 扩展），其他 8 个 B 类方向均使用 `B: rowExt(N)` 格式。格式不一致。 |
| **所在位置** | §3.3，fromMat 6a 9×9 转换矩阵表（L379），源行 Mat4x2、目标列 →Mat4x3 的单元格 |
| **严重程度** | 一般 — 不影响转换规则正确性，仅标签格式不一致 |
| **改进建议** | 将 `rowExt(3)` 修正为 `B: rowExt(3)`，与其他 B 类方向标签格式一致。 |
