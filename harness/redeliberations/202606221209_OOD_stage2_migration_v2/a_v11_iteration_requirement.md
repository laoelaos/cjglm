根据以下审查结果，迭代上一轮的产出，形成新版的文件，从而更好地满足用户需求。

## 当前审查结果

### 问题1（中等）：`fromParts` 家族签名模板的参数数量存在结构矛盾

Mat4xR 族的签名模板固定展示 16 个分量参数，但注释说明"R=2 时 8 个分量、R=3 时 12 个、R=4 时 16 个"，意味着 Mat4x2、Mat4x3、Mat4x4 各自需要不同参数数量的独立签名。仓颉不支持条件编译参数数量，模板信息矛盾无法直接编码。Mat2xR 族的签名模板使用了非法的条件简写语法 `[a20,a21 / a30,a31,a32,a33]`。

**改进建议**：参考 §3.3 item 5 `fromColumns` 的做法，逐一列出 9 个矩阵类型的独立签名，确保参数数量与实际列数×行数一致。或至少在家族模板后补充说明"每个矩阵类型需要独立定义签名，参数数量为 C×R。"

### 问题2（轻微）：`filled(scalar)` 与 `identity(one)` 的 Bool 矩阵排除边界未在声明位置标注

§7 D33 明确记录 Bool 矩阵不提供 `filled()`、`identity()`、`equalEpsilon()`、算术运算符。但 §3.3 item 3 和 item 8 的声明位置没有任何指向 D33 的交叉引用或 Bool 排除注释。

**改进建议**：在 §3.3 item 3 和 item 8 末尾添加指向 D33 的交叉引用注释。

### 问题3（轻微）：`HighpDMat` 方阵短别名命名不一致且未在 §9 记录

HighpDMat 家族的方阵短别名为 `Highpdmat2`/`Highpdmat3`/`Highpdmat4`，与阶段一 Vec 别名的 PascalCase 模式（`HighpDVec2`）不一致。GLM 参考使用 `highp_dmat2`（全小写+下划线）。该命名偏差未被 §9 差异声明记录。

**改进建议**：方案一（推荐）：统一为 `HighpDMat2`/`HighpDMat3`/`HighpDMat4`；方案二：在 §9 新增差异条目记录命名偏差及理由。

### 问题4（轻微）：`fromMat` item 7（同尺寸跨类型）约束条件可能导致下游消费者误解泛型适用范围

§3.3 item 7 声明 fromMat 7 "Bool 矩阵支持此操作"。但目标类型为 Bool 时，conv 闭包实际使用时可能需要比较表达式（如 `{ x => x != 0 }`）而非构造函数调用，文档未就此给出示例或说明。

**改进建议**：在 §3.3 item 7 末尾补充说明：当目标 T=Bool 时，conv 闭包通常需使用比较表达式而非构造函数调用。

### 问题5（轻微）：§8 compile order 的 fallback 方案一（全局自由函数）对 API 调用语法的破坏性变更影响范围未在 §9 记录

§8 已正确分析 fallback 方案一的 API 影响，但该 API 形态的巨大差异未被 §9 差异声明记录。§9 最后一个与行向量×矩阵乘法相关的条目标注为"编译顺序依赖同包延迟解析（待原型验证）"，但未提及方案一的 API 破坏性变更。

**改进建议**：在 §9 对应差异条目末尾添加交叉引用，说明若验证失败将退化为全局自由函数方案，API 形态从 `v * m` 变为 `mul_row_vec_mat(v, m)`。

### 问题6（轻微）：§8 产出物清单中 `lib.cj` 的导出粒度未定义

§8 产出物清单提到"修改文件：src/fwd.cj, src/lib.cj"，但未定义 lib.cj 具体需要新增哪些 public import。§2 也说"lib.cj — 新增矩阵类型的 public import"，但没有给出导入清单。

**改进建议**：在 §8 补充 lib.cj 的 public import 清单，至少明确：(1) 是否导出全部 9 个 detail 矩阵类型；(2) 是否导出 ext/ 别名文件的 public import；(3) 是否导出 scalar_mat_ops.cj 的全局函数。

### 问题7（轻微）：非方阵 `*=` 的简要提及无法指导编码实现

§9 说"非方阵 *= 不可用"，§3.5 的复合赋值回退表也仅对 3 个方阵列出 *=(Mat) 和 /=(Mat)。但对编码实现者而言，此结论意味着非方阵类型（6 个）的 extend 块中 `operator func *=` 可以被安全地省略，文档应明确给出此实现指导。

**改进建议**：在 §3.5 复合赋值回退方案表后添加一句话说明："非方阵不定义 *=(Mat) 和 /=(Mat) 运算符，因乘法结果类型与左操作数类型不一致。"

## 历史迭代回顾

分析历史反馈与当前反馈的关系：

- **已解决的问题**：第 3~9 轮反馈的问题（fromMat 7 conv 参数位置、++/-- 排除、模板赋值运算符、cjpm 子包预验证、B 方向 GLM 对照、fwd.cj 别名清单、stub 签名清单、Vec extend 签名、矩阵-列向量乘法、乘法重载数量、手动定义数量推导、编译顺序分析、填充规则、非方阵 identity、fromMat 非 B 方向 GLM 验证、matrix.cj stub 矛盾、Bool 矩阵 equalEpsilon 排除等）均已在 v17~v21 修订中解决，当前反馈中不再提及。

- **新发现的问题**：本轮 7 个问题中，除问题 1 为持续问题外，其余 6 个（问题 2~7）均为新识别的问题，覆盖交叉引用缺失（问题 2、4、5）、命名一致性（问题 3）、导出粒度未定义（问题 6）、实现指导明确性（问题 7）等维度。
