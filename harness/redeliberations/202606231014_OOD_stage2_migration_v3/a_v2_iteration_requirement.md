## 当前审查结果

1. **[严重] `[]` 赋值版本签名存在双重语法违规**（§3.4 行列访问）：参数名未使用 `value!` 命名参数，且缺少 `mut` 修饰符，将直接导致编译失败。此为历史迭代已修正问题的回归——v1 迭代第 6 轮问题 4 和 v2 迭代第 22 轮已做过类似修正，但当前版本中 `[]` 赋值签名仍缺少 `mut` 和 `value!`。改进建议：将签名修正为 `public mut operator func [](i: Int64, value!: VecR<T,Q>): Unit`，全文检查所有 `[]` 赋值签名之处统一修正。

2. **[中等] 复合赋值运算符自动生成被标注为"待验证假设"**（§3.5 和 §8）：仓颉函数文档 §8.5 明确声明复合赋值为语言规范保证的确定性行为，非待验证假设。当前设计为此"假设"准备了 54+6 个手动定义回退方案表，占据显著文档篇幅且可能误导编码者执行不必要的回退实现。改进建议：(1) 明确声明复合赋值自动生成为语言规范保证行为；(2) 将手动定义回退方案从设计主体移至独立附录并标注"仅用于防御编译器版本回归 bug，非正常编码路径"；(3) 从 §8 编码启动前验证项中删除复合赋值验证项，或将该项降级为可选的非阻断性防御测试。

3. **[中等] 路线图 `type_float.cj` / `type_half.cj` 可选项在设计文档中完全缺席**（全文）：路线图阶段 2 范围中明确列出此可选项，但设计文档未说明纳入也未说明排除，编码者无法确定是否需要实现。改进建议：在 §7 设计决策表中新增一条决策，明确 `type_float.cj` / `type_half.cj` 的阶段归属（纳入/推迟/排除）及理由。

4. **[中等] `lib.cj` 新增导出清单中 `mod` 函数与矩阵标量运算导出的交互未显式说明**（§2 scalar_mat_ops.cj 条目；§8 lib.cj 导出清单）：矩阵不提供 `mod`（§2 已声明），但 `lib.cj` 现有 `mod` 导出仅覆盖向量版，阶段二无新增矩阵 `mod` 重载需导出，设计文档未显式确认此理解。改进建议：在 §8 lib.cj 导出清单中显式说明：(1) `mod` 不在 `scalar_mat_ops.cj` 中提供；(2) `lib.cj` 现有 `mod` 导出仅需覆盖向量版，阶段二无新增修改。

5. **[中等] `common.cj` 中 `abs` 和 `sign` 的泛型约束选用 `Equatable<T>` 而非 `Comparable<T>`**（§3.7 common.cj 签名清单第 585-586 行）：`abs` 和 `sign` 的实现需要 `<`/`>` 比较运算，属于 `Comparable<T>` 接口，仅约束 `Equatable<T>` 无法满足接口需求。`Comparable<T>` 已继承 `Equatable<T>`，改为 `Comparable<T>` 可同时满足且不引入冗余。改进建议：将 `abs` 和 `sign` 的约束从 `where T <: Number<T> & Equatable<T>` 修正为 `where T <: Number<T> & Comparable<T>`。同步检查其余函数约束。

6. **[中等] 矩阵-矩阵除法 `Mat / Mat` 运算符签名完全缺失**（§3.5 算术运算符；§10 API 覆盖矩阵）：方阵的矩阵除法运算符（Mat2x2/Mat3x3/Mat4x4 各一版）从未给出签名，编码者无法确定归属和签名模式。改进建议：在 §3.5 补充方阵矩阵-矩阵除法 `operator func /(rhs: Mat{N}x{N}<T,Q>): Mat{N}x{N}<T,Q>` 的 3 个签名，标注内部实现为 `this * inverse(rhs)`（依赖 inverse stub），泛型约束和 `@OverflowWrapping` 标注与乘法运算符一致。同步更新 §10 API 覆盖矩阵。

7. **[一般] `diagonal(scalar)` 与 `identity(one)` 在方阵上行为重叠，关系未记录**（§3.3 items 3, 8；§7 设计决策表）：当 `scalar = one` 时方阵上两者产生相同结果，v2 迭代第 2 轮问题 4 曾被采纳但当前文档决策表未显式记录。改进建议：(1) 在 §7 决策表中补充决策：方阵 `identity(one)` 内部委托 `diagonal(one)` 实现，非方阵仅提供 `identity(one)`；(2) 在 §4 行为契约的 `identity` 示例旁注明方阵 `identity(1.0f)` 等价于 `diagonal(1.0f)`；(3) 推荐：需单位矩阵时优先使用 `identity`，需一般对角矩阵时使用 `diagonal`。

8. **[一般] `outerProduct` 实现说明中公式 `c * r[j]` 的语义表达存在歧义**（§3.7 matrix.cj）：`c` 的分量数等于行数 R 而非列数 C，命名可能产生"c 的分量数=列数"的直觉误解。改进建议：将公式改写为更精确的元素公式 `M[j][i] = c[i] * r[j]`（i 为行索引，j 为列索引），消除命名歧义。

9. **[提示] 精度前缀方阵短别名的大小写风格可能产生可读性歧义**（§8 fwd.cj 别名清单）：`Highpmat2` 等中 `p` 与 `mat` 拼接处可能被误读。改进建议：若确认保留当前命名，在别名定义处添加注释说明命名规则（精度前缀 + `mat` + 方阵尺寸）。
