## 当前审查结果

### 问题 1（中等）：fromMat 6b T(0) 演算路径仍存在类型系统缺陷（未修复）

§3.3 编码模板说明（第467-472行）仍声称"6a（同类型不同尺寸）与 6b（跨类型不同尺寸）函数体结构一致，仅 6b 签名新增 conv 参数"。此断言存在以下类型系统缺陷：
- 6a 模板中 `let zero = m.c0.x - m.c0.x` 正确计算 T(0)，因为 `m.c0.x` 类型为 T 且 `T <: Number<T>` 已由约束保证。
- 6b 中 `m.c0.x` 类型为 U（源元素类型），`m.c0.x - m.c0.x` 计算的是 U(0) 而非 T(0)，且 6b 约束仅包含 `T <: Number<T>`、未声明 `U <: Number<U>`。若 U 未实现 Number<U>（如 Bool），此代码无法编译。即使 U 实现了 Number<U>，U(0) 的类型是 U 而非 T，无法直接用于构造 VecR<T,Q>。

改进建议：
1. 将 6b 的 T(0) 演算路径从 `m.c0.x - m.c0.x` 修正为 `one - one`（`one: T` 且 `T <: Number<T>`，语义上 1-1=0）。
2. 为 6b 单独提供一个编码模板示例（或至少一段零值计算代码），与 6a 示例并列展示。
3. 若 U 也可能需要参与运算（如 colExtend 中对 U 类型源值应用 conv 前需 U(0)），应增加 `U <: Number<U>` 约束或使用其他方式规避。

### 问题 2（中等）：div(s, m) 语义歧义仍未在设计层面锁定（未修复）

§8 编码启动前验证项（第844行末尾）仍将 `div(s: T, m: Mat)` 的语义标注为"需验证此语义等价性"——"当前 scalar_mat_ops.cj 中 div(s, m) 的实际展开为 m / s 的等价变换——需验证此语义等价性"。具体矛盾：`add(s, m)`、`sub(s, m)`、`mul(s, m)` 的语义一致（标量运算符作用于矩阵各分量），但 `div(s, m)` 若实现为 `m / s`（矩阵各分量除以标量）则与参数顺序暗示的 `s / m`（标量除以各分量）语义相反。

改进建议：在 §7 设计决策表中新增决策项，明确锁定 `div(s, m)` 的最终语义。建议在以下两种方案中选择其一：
- 若设计意图为 `s / m`（标量逐元素除以矩阵），直接给出展开公式 `result[j][i] = s / m.c_j[i]`，移除"待验证"标注。
- 若实际需要 `m / s`（矩阵逐元素除以标量），将函数按正确语义签名——若保持参数顺序 `(s, m)`，则展开公式应为 `s / m[i][j]`；若要表达 `m[i][j] / s`，应将参数交换为 `div(m, s)` 以消除语义错位。

### 问题 3（中等）：§10 API 覆盖矩阵缺少 fromParts/fromColumns/fromMat 系列工厂函数追踪条目（范围扩展）

§10 "mat2x2~mat4x4.hpp 覆盖矩阵"（第1004-1025行）列出了逐分量构造、列向量构造、diagonal、identity 等条目，但以下 5 个工厂函数组（§3.3 items 4-7）完全未在覆盖矩阵中追踪：
- fromParts（跨类型逐分量构造，9 个重载）— item 4
- fromColumns（跨类型列向量构造，9 个重载）— item 5
- fromMat 6a（同类型跨尺寸转换，72 个方向）— item 6a
- fromMat 6b（跨类型跨尺寸转换，72 个方向）— item 6b
- fromMat 7（跨类型同尺寸转换，9 个重载）— item 7

改进建议：在 §10 覆盖矩阵中新增以上 5 个函数组的行级追踪条目，标注实现阶段归属。

### 问题 4（一般）：fromMat 6a/6b 的 `one` 参数 NaN 值行为未在设计层面定义

§3.3 item 6a/6b 的 `one: T` 参数语义说明（第276-282行）定义了 `one` 的类型和选取规则，但未定义当调用方传入 NaN 作为 `one` 时的行为。`identity(one)` 函数存在同样的 NaN 行为未定义问题（第293-318行）。

改进建议：在 §3.3 item 6a/6b 的 `one: T` 参数语义说明和 §3.3 item 8 identity 行为定义中，补充 NaN 值行为约束："调用方应确保 `one` 为非 NaN 值，否则对角填充结果全部为 NaN，行为未定义"。或可定义"当 `one` 为 NaN 时，对角填充自动退化为全 NaN 填充"作为确定性行为。
