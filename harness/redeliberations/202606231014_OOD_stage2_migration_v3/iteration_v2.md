## 需要解决的问题

- **问题描述**：`diagonal(scalar: T)` 仅对方阵提供，非方阵不提供，但 GLM 对所有 9 个矩阵类型均提供 `mat(T scalar)` 构造函数（含非方阵），API 存在对称性缺陷
- **所在位置**：§3.3 item 3 diagonal 构造函数说明（L231-236）；§7 D34（L753）；§9 差异声明 diagonal 条目（L868）
- **严重程度**：一般
- **改进建议**：对所有 9 个矩阵类型均提供 `diagonal(scalar)`，与 `identity(one)` 形成完整配对；若保留当前设计，在 §9 差异声明中明确声明此为仓颉设计与 GLM `mat(T s)` 的行为偏差

- **问题描述**：矩阵-矩阵除法 `Mat / Mat` 在 `rhs` 不可逆（奇异矩阵）时的行为未定义，`inverse` 的奇异矩阵行为策略缺失
- **所在位置**：§3.5 矩阵-矩阵除法签名（L515-521）；§5 错误处理策略（L713）
- **严重程度**：一般
- **改进建议**：在 §5 错误处理策略中新增"矩阵除法奇异矩阵行为"条目，明确 `inverse` 对奇异矩阵的行为选择（抛异常 / 返回未定义值），并在差异声明中相应标注

- **问题描述**：Vec2 `cross`（2D 叉积，属 gtx/exterior_product）未在 §10 追踪，可能导致阶段六实施者遗漏此 API
- **所在位置**：§3.7 geometric.cj 完整函数签名清单（L649）；§10 geometric 函数覆盖矩阵
- **严重程度**：一般
- **改进建议**：在 §10 中为 `cross` 追加备注"Vec2 `cross` 属 gtx/exterior_product，后续阶段六覆盖"

- **问题描述**：`common.cj` 的 `mod` 签名暂定性未在 §8 编码启动前验证项中列为阻断性风险项，`Number<T>` 约束的 `mod` 对 Float32/Float64 为确定性编译期错误
- **所在位置**：§3.7 common.cj mod 签名（L578-579）；§8 编码启动前验证项（L771）
- **严重程度**：一般
- **改进建议**：在 §8 编码启动前验证项中新增阻断项：验证 `mod<T>` 对 Float32/Float64 是否可编译，若不能则拆分签名

- **问题描述**：编码模板示例使用 `m.c0[0]` 下标访问而非 `m.c0.x` 属性访问，将带来不必要的 assert 开销
- **所在位置**：§3.3 编码模板示例 1（L406-423）
- **严重程度**：一般
- **改进建议**：将模板示例中 `[0]`/`[1]` 改为 `.x`/`.y` 属性访问，注明优先使用属性访问的理由

- **问题描述**：`scalar_mat_ops.cj` 新增的 36 个全局函数与 `scalar_vec_ops.cj` 同名函数重载解析未纳入 §8 编码启动前验证项
- **所在位置**：§8 编码启动前验证项（L771）
- **严重程度**：一般
- **改进建议**：在 §8 验证项中增加：验证 `add(s: T, m: Mat2x2<T,Q>)` 与 `add(s: T, v: Vec2<T,Q>)` 重载无歧义

- **问题描述**：`geometric.cj` stub 中缺少 Vec1 版本的 `length`/`normalize`/`distance`/`reflect`/`refract`，与 GLM 之间存在潜在覆盖差异（待查验确认）
- **所在位置**：§3.7 geometric.cj 完整函数签名清单（L650-664）；§10 geometric 函数覆盖矩阵
- **严重程度**：一般
- **改进建议**：查验 GLM `func_geometric.inl` 确认 Vec1 版本是否实际提供特化；若确认遗漏则补齐，若为有意省略则在文档中说明理由

- **问题描述**：`fromMat` 6a/6b 纯收缩方向 `one` 参数被忽略的行为未在 §9 差异声明中显式声明
- **所在位置**：§3.3 item 6 纯收缩方向 `one` 参数行为说明（L286 附近）；§9 差异声明
- **严重程度**：一般
- **改进建议**：在 §9 差异声明中新增条目，声明此为仓颉泛型限制导致的 API 设计偏差
