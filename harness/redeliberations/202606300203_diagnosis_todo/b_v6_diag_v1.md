## 发现的问题

### 问题 1（严重）：G11 真实性分类在正文与汇总表之间矛盾

**所在位置**：第 243-244 行（正文）vs 第 464 行（汇总表）

**问题描述**：
正文将 G11 的"真实性"标注为"真实问题"（第 243 行： "两个同名函数签名不同，造成混淆和使用隐患"），根因标注为"OOD 设计问题"（第 244 行）。但汇总表（第 464 行）将其归入"OOD 设计决策（非问题）"类别——隐含 G11 不属于问题。两条陈述对"G11 是否属于问题"的定性直接矛盾。

正文末尾的"修改方向"（第 252-253 行）建议"不需要改代码，需在 deviations.md 中新增偏差记录"，暗示这不是代码缺陷而是文档/教育范畴。但"真实性"字段的"真实问题"标签与该宽松的修改方向不匹配——若确认为"真实问题"，执行者期望看到修复方案；若仅需文档记录，应将其标注为"接口设计差异"或"非问题（已知设计决策）"，而非"真实问题"。

**改进建议**：
统一分类。两种可选方案：
- 方案 A：将正文"真实性"改为"有争议/非问题"，与汇总表的"OOD 设计决策（非问题）"一致——因为差异源于 GLM 1.0.3 自身 API 设计，非仓颉特有缺陷，仅需 deviations.md 文档记录
- 方案 B：保留正文"真实问题"定性，但修改汇总表分类（移出"非问题"行），新增"接口设计差异"类别

推荐方案 A，更符合证据描述（差异与 GLM 1.0.3 设计一致，仅需文档记录）。

**所在位置**：第 270 行（章节标题"测试覆盖问题（G13-G37 逐条诊断）"）及第 346 行（G24 正文）
