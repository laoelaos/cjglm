## 发现的问题

### P1. [严重] S3 分析中存在事实错误——round-trip 测试实际可捕获 quatCast 因子 2 bug

**位置：** §S3 分析段，第 68-69 行

**问题描述：** 报告声称「由于 quatCast 因子 2 bug 是整体缩放，round-trip 测试无法捕获（缩放因子在往返中自抵消）」。此断言在数学上不成立。对于非身份四元数的 round-trip（如 `testMat3CastNonIdentityQuat`、`testQuatCastNonIdentityMat3RoundTrip`、`testQuatCastNonIdentityMat4RoundTrip`），quatCast 返回的错误四元数分量（非最大分量约 2x）进入 `mat3Cast` 后，矩阵元素涉及分量的二次型（x²、xy 等），导致结果矩阵与原矩阵显著不同，round-trip 断言 `@Expect(m == m2, true)` 将 FAIL。

仅在身份四元数 round-trip 测试中（非最大分量为 0，缩放后仍为 0）才无法捕获该 bug。报告将「部分 round-trip 测试无法捕获」泛化为「所有 round-trip 测试无法捕获」，构成事实错误。

**证据：** 代码 `test_type_quat_cast.cj:56-62` 使用 `q=(0.2,0.3,0.4,0.8)` 的非身份 round-trip。若 quatCast 因子 2 bug 存在，`q1` 的非 w 分量约为 `q` 的 2 倍，`mat3Cast(q1)` 中的 `txx=2*x'²`、`txy=2*x'*y'` 等二次项将被显著改变，矩阵不再相等。

**影响传播：**
- 优先级表第 3 项（S3）理由「若 S1 已修复，本项优先级降为 Low」基于错误前提
- 关键发现汇总第 5 项（S3 条件性降级）同样受影响
- S3 的「测试方法论改进」子建议仍然有效，但核心断言需要修正

**建议：** 修正 S3 分析：区分身份与非身份 round-trip 测试的不同捕获能力；删除「缩放因子在往返中自抵消」的错误表述；在优先级表中移除 S1 修复后 S3 自动降级的假设。

### P2. [中等] 对 deviations.md 的使用方向偏离需求意图

**位置：** §S1~§G6 各节的「deviations.md 对照」段落

**问题描述：** 需求要求「实现偏差注明文档：@docs\deviations.md」用于**交叉验证**——即核对实现行为是否与 deviations.md 记载的偏差一致。但报告将该段落用于**检查注册状态**——即判断当前问题是否已登记至 deviations.md、是否应登记、属于哪类偏差。两者是不同性质的检查。

具体偏离：需求期望的是用 deviations.md 来辅助分类判断（例如：若 deviations.md 已记载某行为偏差，则实现与该行为一致时不应视为「真实存在」）；但报告实际做的是 deviations.md 的元数据管理检查。

**影响：** 需求中明确的 4 个交叉验证文档之一的 deviations.md 未按预期方式使用，可能遗漏某些本可通过 deviations.md 辅助分类判断的问题。

**建议：** 对所有涉及 deviations.md 已记载偏差的现有问题，补充一轮基于 deviations.md 的交叉验证：核实实现行为是否与 deviations.md 记载的偏差一致，而非仅检查是否已登记。

### P3. [中等] S2 修复方案的潜在副作用未评估

**位置：** §S2 分析段及优先级表

**问题描述：** 报告将 S2 列为唯一 Critical 优先级，建议「测试文件命名重构」（`test_*.cj` → `*_test.cj`）。但未讨论以下潜在风险：
- git 历史追溯影响：批量重命名后 `git log --follow` 能否正确追溯
- 测试文件间依赖：若某些测试文件通过相对路径或包名互相引用（如 `tests/glm/detail/test_*.cj` 之间），重命名是否引入编译问题
- CI 配置影响：若 CI 配置中有显式引用 `tests/glm/test_*.cj` 模式的路径，重命名后需同步更新
- 替代方案：是否存在无需重命名所有文件的替代方案（如自定义 cjpm 配置或 wrapper 脚本）

**建议：** 补充 S2 修复方案的风险评估，或至少标注上述待确认项；考虑提供 2-3 种修复选项并对比其 trade-off。
