### v12 修订特点

1. **依赖描述对齐**：本轮核心修复是 §3.7 `length`/§3.9 `axis` 函数依赖描述与 §1「Float32 与 std.math 的交互约束」段的不一致性——该问题在多轮审查中反复出现（第 3/6/11 轮均指向同一根因）。本次全文搜索所有「仅支持 Float64」旧描述的残余引用并替换为与 §1 v11 修订一致的描述。

2. **测试覆盖补齐**：针对测试覆盖的缺口——新增 `gtc/matrix_transform.cj` 64 个 stub 的测试覆盖（`test_matrix_transform_stubs.cj`，≥8 用例），新增「已实现但被 stub 阻塞」函数覆盖类别（第 8 类），统一 snake_case→camelCase 命名。

3. **验证优先级明确**：针对 `FloatingPoint<T>` 接口可用性验证长期滞后的风险，新增「高优先级编译前验证项」子节，将两项基础性编译假设验证列为 P0，要求在编码阶段前完成最小仓颉测试文件验证。

4. **用例数统一**：消除 `test_ext_scalar_constants.cj` 用例数计算的多重矛盾，按统一口径计算为 ≥13。

### v13 修订特点

1. **权威来源完整性补齐**：本轮最关键的修复是 §11.5 函数可用性对照表——此前缺失 7 个关键函数条目（导致下游按表核验时遗漏覆盖），本轮一次性补齐。同时 `fromMat3`/`fromMat4` 的约束传递性注记使表不再隐藏"内部调用携带约束"的信息。

2. **公式与 §1 策略对齐**：`axis` 公式移除 `Float64(tmp1)` 冗余转换，使实现公式与 §1 方案 A 完全一致——当 T=Float32 时 `std.math.sqrt` 直接接受 Float32 参数返回 Float32；当 T=Float64 时接受 Float64 参数返回 Float64；不再产生「§1 说直接调用但公式却写显式转换」的自相矛盾。

3. **验证项去重**：消除 §8 编码启动前验证项中长期存在的重复（验证项 20/29 与 25/30），将独立验证目标从声称的 30 项修正为实际 28 项。

4. **rotate 状态画像统一**：在 §11.5（可用性）、§11.6（可达性）、§8（产出物分类）三处同步补充 `rotate` 的 stub 状态标注与说明，消除下游一次性获取状态画像的障碍。

5. **测试可追溯性**：新增「用例到函数的逐项分配依据表」，使 ≥199 用例的构成可逐函数追溯，消除约 15 个用例"无源可查"的问题。

### v23 修订特点

1. **虚假声称系统性清除**：本轮最核心的修复是删除全部 8 个「审查意见修复率累计达 100%」的虚假声称。该问题根因是文档缺乏诚实记录未解决问题能力的系统性缺陷——本轮通过 §8.3 I 节「已知未解决问题列表」从根源上建立诚实的质量记录机制，后续轮次不得再声称"100% 修复率"除非 I 节为空。

2. **§11.5 权威性恢复**：v20 因缺少 5 个 API 条目自降级为「主要参考来源」后，历经 v21-v22 两轮补齐条目但未声明升级。本轮在确认 v22 已完整覆盖全部阶段三 API 后，将权威声明升级回「单一权威来源」，同时 §8.3 D 节引用措辞同步对齐。

3. **版本号核验机制化**：H3 新核验项的引入使版本号一致性的检查从「人工校对」升级为「grep 自动化」，阻断同一根因问题（版本号混乱）在第 10→14→15→17→18→22 轮的反复出现。

4. **签名冻结纪律建设**：通过审批流程 + 核验项的双重机制，将「签名不变」从口头承诺升级为有约束力的设计纪律。

5. **索引覆盖全面化**：§11.7 从仅覆盖 ✅ 函数扩展为全函数覆盖（✅/⚠️/❌），消除了约 88 个 ⚠️/❌ 函数无集中索引的信息碎片化问题。

6. **修复责任制明确**：全部 29 项验证项补充「修复责任人」列后，验证失败后的修复归属从模糊的「设计者/编码者协商」变为逐项明确的职责指派。

7. **核验流程可执行性提升**：H1 关联自动化工具、H 节新增核验执行者与周期声明、§9a 与 §10 合并降冗余——三项措施使设计文档的自核验能力从「仅文档描述」升级为「可执行流程」。

### v24 修订特点

1. **const 推理概念纠偏**：本轮最核心的修复是纠正 §5.4 中 `inverse` const 拒绝理由的概念混淆。原有推理将「运行时抛异常」作为 const 拒绝理由，这是对 const 函数约束语义的根本性误读——const 函数约束是关于编译器能否在编译期对函数求值，而非关于函数运行时是否可能抛异常。修订后使用「整数除法非 constexpr 兼容」的准确表述。

2. **策略层逻辑一致性修复**：§1 `T(Float64(n))` 整型支持声明与 §3.12 scalar_constants 整型拒绝策略的自相矛盾历经 6 轮（迭代 9-10、迭代 23-24）未闭环，根源在于设计方始终假设整数 `T(Float64(n))` 编译可行但未验证。本轮通过限定支持声明的类型范围、降级为 P0 假设、增加回退路径三管齐下，消除策略层面矛盾。

3. **依赖标注语义精确化**：trigonometric.cj 表的依赖标注从「声称是当前依赖」修正为「阶段四依赖」，消除「函数体为 throw 却声称依赖 std.math」的语义矛盾。

4. **边界条件覆盖系统性补齐**：本轮新增的 5 类边界条件（NaN/Inf 传播、lerp 非有限输入、单位/零矩阵、axis NaN w、inverse 非单位输入）补齐了自迭代 1 起持续报告的覆盖缺口。当前 §5.3 表共覆盖 24 类边界场景。

5. **权威声明精确校准**：§11.5 定位从「单一权威来源」调整为「函数级权威来源，与 §9a/§10 互补」——承认 §9a 的 GLM 基准粒度（按头文件聚合）和 §10 的覆盖状态视图（按文件分组）各有独立的不可替代价值，与 §11.5 的逐函数精细追踪形成多层次覆盖体系。
