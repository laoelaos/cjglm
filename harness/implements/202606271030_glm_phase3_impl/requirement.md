# GLM Phase 3 审议式实现

## 任务概述
基于 OOD 设计文档，实现 GLM 项目的 Phase 3 功能。

## 输入文档

### OOD 设计
- **主设计文档**: `docs/05_ood_phase3.md` — 本次需要实现的 OOD 设计
- **前序 OOD**: `docs/` 目录下其他 `*_ood_*.md` 文件，用于了解上下文
- **路线图**: `docs/` 目录下的路线图文档

### 参考项目
- `references/glm-1.0.3/` — 参考实现，可参照其设计模式

### 项目结构
- **项目根目录**: `cjglm/` — 仓颉项目根目录
- 所有源码和测试直接写入 `cjglm/` 目录树

### 项目实施偏差
- **偏差文档**: `docs/deviations.md` — 记录因仓颉语言限制导致的实施偏差
- 偏差需要在实现过程中根据偏差文档开头的指示添加到相应位置

## 特殊要求
1. Plan Agent 需要在 `plan.md` 开头添加详细的实施路线表格，之后每次进展打勾确认
2. 实现过程中如遇到因仓颉限制无法按 OOD 设计实现的情况，需按 `docs/deviations.md` 开头的指示记录偏差
