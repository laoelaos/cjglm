## 发现

### [严重] 测试方案无法验证 ±0 符号位保留

任务要求添加 `@Expect(roundPowerOfTwo(-0.0), -0.0)` 等测试验证负零符号位保留，但 `@Expect` 底层使用 `==` 比较，而 IEEE 754 规定 `-0.0 == 0.0` 为 true。这意味着即使实现代码未做任何修改（仍返回 `+0.0`），测试也会通过，无法起到验证作用。

需要改用能区分 ±0 的验证手段，例如：
- 位模式比较：`(roundPowerOfTwo(-0.0) as Float64).getOrThrow().toBits() == (-0.0).toBits()`
- 取倒数符号检测：`Float64(1.0) / roundPowerOfTwo(Float64(-0.0)) == -Float64.infinity()`（因 `1.0 / +0.0 = +Inf`，`1.0 / -0.0 = -Inf`）

### [一般] plan.md 路线表未规划本条更新

原需求第 3 条要求"每轮进展打勾确认"，plan.md 的路线表目前只有 v1-v6 列。P2-5（G9）在所有轮次列中均无标记，本轮任务未包含将路线表更新（新增 v7 列或标记 P2-5 完成）的步骤。
