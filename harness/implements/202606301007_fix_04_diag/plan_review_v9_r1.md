# 计划审查报告（v9 r1）

## 审查结果
REJECTED

## 发现

- **[一般]** G17 三角恒等式测试缺少容差处理。方案要求用 `@Expect` 精确对比 `sin²(a) + cos²(a) == 1.0`、`tan(a) == sin(a)/cos(a)` 等恒等式。对于大部分角度值，因 IEEE 754 浮点舍入误差，`sin²(a) + cos²(a)` 的结果不会精确等于 1.0（误差约 1-3 ULP），`@Expect` 精确相等断言会失败。例如 `pi<Float64>()/4.0` 并不是精确的 π/4，sin/cos 返回值不会精确为 √2/2，平方和不会精确为 1.0。需指定容差比较方式，如 `@Expect(abs(sin(a)*sin(a) + cos(a)*cos(a) - Float64(1.0)) < Float64(1e-15), true)`，或选择确保恒等式精确成立的特殊角度。

- **[轻微]** G16 描述"使 Float32 测试深度与 Float64 版本对齐（2-3 组值含边界）"虽可通过对照 Float64 测试来推导，但可更明确。建议对每个函数组给出示例性的期望增量。

- **[轻微]** 计划未提及 deviations.md 更新检查。虽然 v9 变更均为测试文件修改，不太可能产生新偏差，但需求第 2 条要求关注此事项，应在计划中做出说明。

## 修改要求

1. **G17 三角恒等式测试**：在 task_v9 或补充说明中明确使用容差比较方式，避免 `@Expect` 精确相等因浮点误差失败。建议对 `sin²+cos²=1` 使用 `abs(sin²+cos² - 1.0) < epsilon` 形式，对 `tan = sin/cos` 使用交叉相乘验证 `sin - tan*cos ≈ 0`。
