## 发现

### **[严重]** P1-4（roundEven）：负半整数输入产生错误结果

**问题**：设计的 `if (rInt % 2 == 0) { r } else { r - one }` 分支在输入为负半整数时给出错误结果。

**示例**：
| 输入 | roundT | rInt | rInt%2 | 分支 | 设计结果 | 正确 roundEven |
|------|--------|------|--------|------|---------|---------------|
| -2.5 | -3 | -3 | -1(≠0) | else: r-1 | **-4** | -2 |
| -0.5 | -1 | -1 | -1(≠0) | else: r-1 | **-2** | 0 |
| -4.5 | -5 | -5 | -1(≠0) | else: r-1 | **-6** | -4 |

**原因**：仓颉 `%` 为 remainder（符号与被除数一致），`-3 % 2 = -1 ≠ 0` 进入 else 分支。而 `r - one` 对负数变成"离零更远"而非"向零调整"，导致选择了错误的相邻偶数。

**对比 GLM 1.0.3 实现**：GLM 对 .5 情况使用 trunc-toward-zero 提取整数部分（而非 away-from-zero），当整数部分为奇数时向无穷方向调整一位。当前设计方案仅在正数时等效。

**修正方向**：else 分支需要根据符号区分：
```cangjie
if (rInt % 2 == 0) {
    r
} else {
    if (r >= zero) { r - one } else { r + one }
}
```

或等价地使用 `sign` 辅助：
```cangjie
if (rInt % 2 == 0) { r } else { r - sign(r) * one }
```
