# 设计审查报告（v8 r4）

## 审查结果

## 发现

### **[严重]** B.3 节 matrix_access.cj：非方阵 row/column 函数签名全部系统性错误

**文件行**：`detail_v8.md` 第 910–915 行（row 签名）、第 980–981 行（column 签名）、第 943–951 行（实现描述）

**问题描述**：
cjglm 的矩阵命名约定为 `Mat{Cols}x{Rows}`（已验证：`Mat2x3` 有 `c0: Vec3`、`c1: Vec3` → 2 列 3 行）。基于此约定，`row` 的返回维度应等于矩阵列数（每行有"列数"个分量），`column` 的返回维度应等于矩阵行数（每列有"行数"个分量）。

但设计中涉及非方阵的 **所有 6 个 `row` 重载** 和 **2 个 `column` 重载** 的签名全部反置。此外，第 943–951 行的实现描述不仅签名反置，还访问了不存在的字段（如 `Mat2x3` 只有 `c0`、`c1`，但实现访问 `m.c2.x`），将导致编译失败。

具体错误：

| 函数签名（设计稿） | 应修正为 | 原因 |
|---|---|---|
| `row(m: Mat2x3, ...): Vec3` | `Vec2` | Mat2x3 有 2 列，每行 2 个分量 |
| `row(m: Mat3x2, ...): Vec2` | `Vec3` | Mat3x2 有 3 列，每行 3 个分量 |
| `row(m: Mat2x4, ...): Vec4` | `Vec2` | Mat2x4 有 2 列 |
| `row(m: Mat4x2, ...): Vec2` | `Vec4` | Mat4x2 有 4 列 |
| `row(m: Mat3x4, ...): Vec4` | `Vec3` | Mat3x4 有 3 列 |
| `row(m: Mat4x3, ...): Vec3` | `Vec4` | Mat4x3 有 4 列 |
| `column(m: Mat2x3, ...): Vec2` | `Vec3` | Mat2x3 有 3 行，每列 3 个分量 |
| `column(m: Mat3x2, ...): Vec3` | `Vec2` | Mat3x2 有 2 行，每列 2 个分量 |

第 943–951 行的实现描述需要同步修正，核心原则：
- `row(m: Mat{C}x{R}, i)` → `Vec{C}`，从 `m.c[0..C].field[i]` 构造
- `column(m: Mat{C}x{R}, i)` → `Vec{R}`，等价于 `m.c[i]`

另外第 943–944 行实现（`Vec3(m.c0.x,m.c1.x,m.c2.x)`）访问了 `Mat2x3` 不存在的 `c2` 字段，属于编译错误级别的缺陷。

## 修改要求（仅 REJECTED 时）

**[严重]** 问题：B.3 整节需要重新设计 row/column 的非方阵重载签名，修正所有 6 个 row 和 2 个 column 的返回类型，并同步修正实现描述中的字段说明。建议设计者对照实际矩阵类型的列向量维度和字段数量进行逐类型验证。修正方向：
1. `row(m, i)` 返回向量维度 = 矩阵列数（即 `Mat{Cols}x{Rows}` → `Vec{Cols}`）
2. `column(m, i)` 返回向量维度 = 矩阵行数（即 `Mat{Cols}x{Rows}` → `Vec{Rows}`），实现等价于 `m.c[i]`
3. 第 943–951 行的实现代码需全部对照验证，确保不访问不存在的字段
